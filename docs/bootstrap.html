§1 — What we are building

Helix is a long-horizon agent orchestrator: a dumb outer loop with three disciplined phases — plan, implement, judge — wrapped around a smart native worker. The loop holds no model judgment; it composes prompts from files, invokes the worker, runs a tiered completion oracle, enforces caps and gates, and persists state. The durable record — plain-text artifacts plus git history — is the asset, not any single agent session.

The markdown source of truth for this design lives at docs/specs/2026-06-30-helix-bootstrap-design.md. This page is the rendered human lens over it; the markdown wins on any disagreement.

§2 — The loop

§3 — Fixed design choices

  1. Dumb orchestration, smart worker. No model judgment in the loop; never reimplement the worker's tool surface (model-harness fit).
  2. Phase separation, fresh context. Plan / implement / judge hand off through files; the judge is always a separate invocation.
  3. Markdown-and-files as truth; HTML as a rendered lens.
  4. The human's seat is the plan phase. The agreed plan is a signed contract.
  5. A tiered completion oracle. Fast surrogate checks each iteration; slow / physical blocking gates pause for human ground truth.
  6. Provenance and session chaining are first-class. Self-contained, human-sortable sessions; age- and condition-stamped findings.
  7. Parallel where idempotent, serial at the gate.
  8. Project-agnostic core, project-specific data. Drift is the chief long-term risk and is designed against. reckon is composed, never absorbed.

§ Decisions

What license should Helix carry?

MIT — permissive and open. CC BY-ND was considered and rejected: NoDerivatives is the wrong fit for code meant to be extended. reckon is on MIT too.

Package layout — src/ or flat repeated-name?

Flat helix/helix/ to match sibling repos; src/ only buys import-isolation we do not need.

How are state record types defined?

LinkML ( schema/helix.yaml ) is the single source of truth; pydantic v2 models generated via gen-pydantic into helix/models.py (not hand-edited).

How is the plan tracked?

Markdown design doc is the source of truth; this reckon HTML plan is the rendered lens. Helix core imports nothing from reckon.
